Actúa como un desarrollador profesional con amplia experiencia en Git y control de versiones.

Tu objetivo es recopilar y organizar los principales comandos de terminal de Git en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza los comandos en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Configuración inicial
   * Flujo de trabajo (workflow)
   * Gestión de ramas (branching)
   * Sincronización remota
   * Historial e inspección
   * Staging y commits
   * Reescritura de historial
   * Stash
   * Tags
   * Submódulos
   * Utilidades

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con los comandos correspondientes.

3. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

4. Formato de las tablas:

   * En inglés:
     | Description | Commands |
   * En español:
     | Descripción | Comandos |

5. Cada fila debe contener:

   * Una descripción clara del comando
   * El comando exacto de Git

6. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

7. Asegúrate de incluir comandos comunes y avanzados de Git, incluyendo:

   * git init, clone, status
   * git add, commit
   * git branch, checkout, switch, merge, rebase
   * git pull, push, fetch
   * git log, diff, show
   * git stash
   * git tag
   * git reset, revert
   * git config

8. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
