Actúa como un desarrollador profesional con amplia experiencia en Docker y herramientas de contenedorización.

Tu objetivo es recopilar y organizar los principales comandos de terminal de Docker en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza los comandos en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Instalación y configuración
   * Gestión de imágenes
   * Gestión de contenedores
   * Redes
   * Volúmenes
   * Docker Compose
   * Inspección y monitoreo
   * Limpieza y mantenimiento
   * Utilidades avanzadas

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con los comandos correspondientes.

3. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

4. Formato de las tablas:

   * En inglés:
     | Description | Commands |
   * En español:
     | Descripción | Comandos |

5. Cada fila debe contener:

   * Una descripción clara del comando
   * El comando exacto de Docker

6. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

7. Asegúrate de incluir comandos comunes y avanzados de Docker, incluyendo:

   * docker run, build, pull, push
   * docker ps, logs, exec, inspect
   * docker network, volume
   * docker compose

8. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
