Actúa como un desarrollador profesional con amplia experiencia en sistemas Windows y uso avanzado de la línea de comandos (CMD y PowerShell).

Tu objetivo es recopilar y organizar los principales comandos de Windows en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza los comandos en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Navegación del sistema de archivos
   * Gestión de archivos y directorios
   * Procesos y rendimiento
   * Redes
   * Usuarios y permisos
   * Información del sistema
   * Gestión de discos
   * PowerShell (comandos clave)
   * Administración del sistema
   * Utilidades avanzadas

2. Considera comandos tanto de:

   * Símbolo del sistema (CMD)
   * PowerShell

3. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con los comandos correspondientes.

4. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

5. Formato de las tablas:

   * En inglés:
     | Description | Commands |
   * En español:
     | Descripción | Comandos |

6. Cada fila debe contener:

   * Una descripción clara del comando
   * El comando exacto (indicando si es CMD o PowerShell cuando sea necesario)

7. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

8. Asegúrate de incluir comandos comunes y avanzados, tales como:

   * dir, cd, cls
   * copy, move, del
   * tasklist, taskkill
   * ipconfig, ping, netstat
   * systeminfo
   * chkdsk, diskpart
   * net user
   * Get-Process, Get-Service
   * Set-ExecutionPolicy
   * Get-Help

9. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
