Actúa como un desarrollador profesional con amplia experiencia en **Visual Studio Code**, incluyendo su uso desde la **consola/terminal** en diferentes sistemas operativos.

Tu objetivo es recopilar y organizar los **comandos más recurrentes y útiles de VS Code desde la CLI** en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza los comandos en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Apertura de archivos y carpetas
   * Navegación
   * Extensiones
   * Configuración y ajustes
   * Integración con Git
   * Depuración
   * Utilidades avanzadas (tasks, snippets, workspace management)
   * Atajos y flags de la CLI

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con los comandos correspondientes.

3. Para cada comando:

   * Incluye una descripción clara de su propósito
   * Muestra el comando exacto con sintaxis completa
   * Indica parámetros importantes cuando corresponda

4. Debes generar dos versiones completas del contenido:

   * Una en **español**
   * Una en **inglés**

5. Formato de las tablas:

   * En inglés:
     | Description | Command |
   * En español:
     | Descripción | Comando |

6. Cada fila debe contener:

   * Una descripción clara y profesional
   * El comando exacto

7. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

8. Asegúrate de incluir comandos comunes y avanzados de la CLI de VS Code, tales como:

   * Abrir archivos y carpetas: `code <file/folder>`
   * Abrir en ventana nueva: `code -n <file/folder>`
   * Abrir en modo solo lectura: `code -r <file/folder>`
   * Listar extensiones: `code --list-extensions`
   * Instalar extensión: `code --install-extension <ext>`
   * Desinstalar extensión: `code --uninstall-extension <ext>`
   * Ejecutar en modo verbose: `code --verbose`
   * Diferentes flags de configuración: `--disable-extensions`, `--wait`, `--user-data-dir`
   * Integración con Git: abrir repositorios, cambios, etc.
   * Ejecutar tareas desde CLI: `code --task <taskName>`

9. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
