Actúa como un desarrollador profesional con amplia experiencia en Java y Spring Boot, incluyendo desarrollo backend y arquitectura de aplicaciones.

Tu objetivo es recopilar y organizar las principales anotaciones de Java y Spring Boot en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza las anotaciones en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Anotaciones básicas de Java
   * Anotaciones de Spring Core
   * Inyección de dependencias
   * Configuración
   * Web / REST
   * Persistencia (JPA / Hibernate)
   * Validación
   * Seguridad
   * Testing
   * Lombok (si aplica)
   * Utilidades avanzadas

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con las anotaciones correspondientes.

3. Para cada anotación:

   * Incluye una descripción clara de su propósito
   * Muestra la anotación exacta (ej: @Component, @Autowired)

4. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

5. Formato de las tablas:

   * En inglés:
     | Description | Annotation |
   * En español:
     | Descripción | Anotación |

6. Cada fila debe contener:

   * Una descripción clara y profesional
   * La anotación exacta

7. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

8. Asegúrate de incluir anotaciones comunes y avanzadas, tales como:

   * @Override, @Deprecated
   * @Component, @Service, @Repository, @Controller
   * @Autowired, @Qualifier
   * @Configuration, @Bean
   * @RestController, @RequestMapping, @GetMapping, @PostMapping
   * @Entity, @Id, @GeneratedValue, @Column
   * @Valid, @NotNull, @Size
   * @SpringBootTest, @MockBean
   * @EnableAutoConfiguration
   * @Data (Lombok)

9. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
