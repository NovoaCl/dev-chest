Actúa como un desarrollador profesional con amplia experiencia en Java, incluyendo conocimiento profundo de todas sus versiones desde Java 1.0 hasta la última versión estable.

Tu objetivo es recopilar y organizar todos los conceptos y funcionalidades de Java en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza los conceptos y funcionalidades en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Fundamentos de Java (Variables, Tipos de datos, Operadores)
   * Control de flujo (if, switch, loops)
   * Programación orientada a objetos (Clases, Objetos, Herencia, Polimorfismo, Encapsulación, Abstracción)
   * Colecciones y Generics
   * Excepciones y manejo de errores
   * Entrada/Salida (I/O)
   * Concurrencia y multihilo
   * Java Streams y Lambda expressions
   * Funcionalidades específicas por versión (ejemplo: Java 8 Streams, Java 9 Modules, Java 14 Records, Java 17 Sealed Classes, etc.)
   * APIs comunes (Collections, Date/Time, Networking, JDBC)
   * Características avanzadas (Reflection, Anotaciones, Serialización, Seguridad)
   * Utilidades y librerías estándar
   * Testing y herramientas (JUnit, Mockito)

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con los conceptos o funcionalidades correspondientes.

3. Para cada concepto o funcionalidad:

   * Incluye una descripción clara de su propósito
   * Muestra sintaxis, ejemplos de uso o notas relevantes
   * Indica si corresponde a una versión específica de Java cuando aplique

4. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

5. Formato de las tablas:

   * En inglés:
     | Description | Feature / Concept | Example / Notes |
   * En español:
     | Descripción | Funcionalidad / Concepto | Ejemplo / Notas |

6. Cada fila debe contener:

   * Una descripción clara y profesional
   * El concepto o funcionalidad exacta
   * Ejemplo o notas cuando sea relevante

7. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

8. Asegúrate de incluir funcionalidades y conceptos de todas las versiones relevantes de Java, incluyendo:

   * Java 1.x hasta la última versión estable
   * Funcionalidades introducidas en cada versión, como: Lambdas y Streams (Java 8), Modules (Java 9), Local-Variable Type Inference (Java 10), Text Blocks (Java 13), Records y Pattern Matching (Java 14-16), Sealed Classes (Java 17), etc.

9. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
