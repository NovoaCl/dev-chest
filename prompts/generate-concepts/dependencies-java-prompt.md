Actúa como un desarrollador profesional con amplia experiencia en Java, Spring Boot y gestión de dependencias con Maven y Gradle.

Tu objetivo es recopilar y organizar las principales dependencias utilizadas en proyectos Java y Spring Boot en una estructura clara, completa y profesional.

### Requisitos:

1. Organiza las dependencias en secciones lógicas basadas en buenas prácticas, por ejemplo:

   * Dependencias base de Spring Boot
   * Web / REST
   * Seguridad
   * Persistencia (JPA / Hibernate)
   * Bases de datos
   * Validación
   * Testing
   * Documentación (Swagger / OpenAPI)
   * Logging
   * Utilidades
   * Lombok
   * DevTools y desarrollo

2. Para cada sección:

   * Incluye una breve descripción de la categoría.
   * Crea una tabla con las dependencias correspondientes.

3. Para cada dependencia:

   * Incluye una descripción clara de su propósito
   * Muestra la dependencia en formato:

     * Maven (XML)
     * Gradle (Groovy o Kotlin DSL si aplica)

4. Debes generar dos versiones completas del contenido:

   * Una en inglés
   * Una en español

5. Formato de las tablas:

   * En inglés:
     | Description | Dependency |
   * En español:
     | Descripción | Dependencia |

6. Cada fila debe contener:

   * Una descripción clara y profesional
   * El bloque de dependencia correspondiente (Maven y Gradle)

7. El resultado final debe:

   * Estar completamente consolidado en un único archivo en formato Markdown (.md)
   * Tener una estructura clara, profesional y fácil de navegar
   * No incluir ningún emoji ni símbolos innecesarios

8. Asegúrate de incluir dependencias comunes y relevantes, tales como:

   * spring-boot-starter-web
   * spring-boot-starter-data-jpa
   * spring-boot-starter-security
   * spring-boot-starter-test
   * spring-boot-devtools
   * lombok
   * mysql-connector-j / postgresql
   * h2
   * springdoc-openapi
   * logback / slf4j

9. Mantén consistencia en estilo, redacción y formato entre ambas versiones (inglés y español).
